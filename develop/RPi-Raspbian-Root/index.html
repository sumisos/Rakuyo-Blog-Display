「开发」树莓派安装Raspbian系统并启用root用户

发表于 2018-11-25 更新于 2020-03-02 分类于 开发
本文字数： 1.6k 阅读时长 ≈ 3 分钟

准备工作

一台正常工作的笔记本电脑（我的是Win10）
一块正常工作的树莓派（我的是3B+）
一张正常工作的TF卡（我的是32G class 10）
官方Raspbian纯净版镜像「RASPBIAN STRETCH LITE」
专门的SD卡格式化工具SDFormatter「SD Memory Card Formatter」
广受好评的Windows烧录工具Win32DiskImager「Win32 Disk Imager」
（本人）唯一指定win平台ssh工具Putty「PuTTY」

烧录系统

  1. 以防万一用SDFormatter初始化TF卡
  2. 然后用Win32DiskImager把之前下的「2018-11-13-raspbian-stretch-lite.img」写入TF卡
  3. 然后win10会重新读取TF卡，会发现有两个盘，其中一个盘无法识别，提醒你是否格式化

    此时（烧录系统完成后）不要点格式化
    不然就白烧录了，linux系统的磁盘分区win读不出来很正常

  4. 去windows可以读的那个只有几十M的boot盘的根目录下新建一个文件，命名为ssh（没有后缀名）

    某次更新后raspbian为了安全起见默认禁用ssh，但可以手动打开
    系统初始化检测到根目录下有名为ssh文件会删除此文件并启用ssh

ssh连接

  1. 把TF卡插入树莓派（卡槽在板子背面），用网线连接笔记本和树莓派，接通电源。
  2. Win+X，打开PowerShell(win10的集成版cmd)，执行arp -a，一般情况会有一个MAC地址b8开头的动态IP，那个就是树莓派IP

    如果没有我也没办法，具体情况具体解决
    我连的时候也没找到，我用「Advanced IP Scanner」扫的
    而且扫出来Manufacturer（制造商）为Raspberry Pi Foundation一堆，最多的时候有8个OTZ
    但是根据观察笔记本的Name后面带了.mshome.net的后缀
    我寻思同理找到raspberrypi.mshome.net那项应该就是树莓派真正的IP了

  3. 打开putty执行树莓派IP，ssh连接，用pi/raspberry的初始密码登录
  4. 末行显示pi@raspberrypi:~ $，成功登录，使用ifconfig wlan0可以看到IP

启用root用户

  1. 执行sudo passwd root设置root密码并牢记，忘记密码自行解决

  2. 执行sudo passwd --unlock root解锁root用户

  3. 执行su root切换到root用户，然后执行第1步设置的密码

  4. 执行apt-get install -y vim下载vim

    因为纯净的镜像只有nano，如果你会用也可以不下，我反正用不惯nano

  5. 然后重点来了，执行vim /etc/ssh/sshd_config编辑ssh配置

  6. 找到#PermitRootLogin prohibit-password这行，意思是允许root登录，但禁止用密码登录，前面有#号说明这行被注释掉了

  7. 不用动被注释的这行，在下面一行加上PermitRootLogin yes，然后按ESC退出到命令模式，再执行:wq保存并退出

  8. 执行/etc/init.d/ssh restart重启ssh

  9. 现在就可以用ssh登录root用户了

修改pi用户

  1. 玩过树莓派都知道pi/raspberry这个默认账户，为了提高安全性我们把它改掉

    这里我是把用户名改成rakuyo，具体你想改什么自行替换命令

  2. 断开会话，新建一个会话用root登录，确保修改的时候pi这个账户没人使用

  3. 执行usermod -l rakuyo pi修改用户名

  4. 执行groupmod -n rakuyo pi修改用户组

  5. 执行mv /home/pi /home/rakuyo 修改home目录

  6. 执行usermod -d /home/rakuyo rakuyo设置home目录

或者直接userdel -r pi删除pi用户，反正以后都用root用户登录

注意

此后的我写的所有树莓派命令都默认有root权限，因此不再加sudo前缀
